# 콘텐츠 밀도/시뮬레이터 사용성 진단 보고서

## 1) 문제 요약
- **교육 설명량이 부족**: 대부분의 페이지가 정의/간단 요약 수준에서 멈추고, 실제 학습 흐름(설명 → 예제 → 구현 → 문제풀이)이 충분히 이어지지 않음.
- **시뮬레이터 사용성 저하**: 코드 실행은 가능하지만 기본 코드에 `print()`가 거의 없어 터미널 피드백이 비어 있음.
- **커스텀 코드 학습 난이도**: 유저가 자유롭게 코드를 수정할 수 있으나,
  - 시각화는 **Adapter 변수명 규약** 또는 **Tracer 이벤트**에 의존하여
  - 규약을 모르면 시각화가 나오지 않거나 제한적으로만 반영됨.

---

## 2) 현 상태(근거 요약)
1) **Code Editor는 커스텀 가능**
   - `CTPPlayground`에서 코드 편집이 가능하며, 실행 시 `use-skulpt-engine`이 Code → Step → Visualizer로 연결됨.
2) **터미널 출력은 print 기반**
   - `CTPTerminal`은 `stdout` 배열을 출력함.
   - 기본 코드 `print()` 표준화 적용으로 출력 피드백 개선됨.
3) **시각화 연결 조건**
   - Linear DS: Adapter 변수명 규약(예: arr, queue, front/rear 등)에 의존
   - Graph/Trie/UF/Algorithms: Trace 이벤트 기반 (`trace()` 호출)으로 시각화 강화
4) **기본 코드 구성**
   - 알고리즘 코드 외에 **시각화 연결용 상태 변수**가 포함됨
   - (예: `compare_indices`, `active_index`, `heap_size`, `swap_indices`)
   - 목적은 “코드 실행 → 즉시 시각화 반영”을 보장하기 위함
5) **알고리즘 코드 전용 편집**
   - 기본 코드에서 시각화/출력 전용 블록을 완전히 숨김
   - 사용자는 알고리즘 코드만 보고 수정 가능

---

## 3) 학습 효과 영향
- 설명량 부족 → **개념-구현-문제 연결이 끊김**
- print 출력 부족 → **실행 결과가 눈에 보이지 않음**
- 규약 미인지 → **커스텀 코드 학습이 실패**로 느껴짐

---

## 4) 개선 방향 (권장)

### A. 콘텐츠 밀도 강화 (교육형 문서 구조)
각 Topic에 최소 아래 섹션을 고정:
1) 개념 정의 + 불변식
2) 직관 예시 (단계별 스냅샷)
3) 구현 포인트 (핵심 로직/의사코드)
4) 실습 예제 (입력/출력 설명 포함)
5) 코딩테스트 문제 연결 (Baekjoon)

### B. 기본 코드 템플릿 표준화
모든 기본 코드에 다음 최소 항목 추가:
- **입력값 지정**
- **핵심 연산 함수 호출**
- `print()`를 통한 결과 출력
- (가능하면) Trace 이벤트 포함

### C. 커스텀 코드 가이드 강화
1) **Adapter 변수명 가이드**를 Playground 상단에 짧게 노출
2) “시각화가 안 보이면 확인할 것” 체크리스트 제공
3) 기본 코드에 **규약을 보여주는 주석** 포함

## 8) DFS/BFS 분리 및 알고리즘 목차 확장(검토 결과)
### 판단 요약
- **Graph 챕터는 구조/표현/인접 리스트/사이클**에 집중시키고,
  DFS/BFS는 **간단 개념 확인용**으로 분리 운영하는 것이 학습 부담을 낮춤.
- **알고리즘 챕터의 DFS/BFS는 코딩테스트 대비형**으로 재구성하는 것이 효과적.

### Graph(비선형) 쪽 구성 제안
- DFS와 BFS를 **분리된 sub-concept**로 유지
- 내용 범위: “탐색 개념/방문 순서/스택 vs 큐 차이” 정도로 축소
- 예제 그래프는 **단순 구조(6~8 노드)**로 고정
- 시각화는 **계층형 레이아웃 + 방문 순서 강조**에만 집중

### Algorithms 쪽 DFS/BFS 목차 확장안(예시)
#### DFS 계열
1) DFS 기본 (재귀/스택, 방문 순서)
2) **백트래킹 패턴** (부분해/가지치기)
3) **트리에서의 DFS** (전위/중위/후위)
4) **사이클/방문상태 패턴** (색칠법)
5) **경로 탐색/경로 복원** (parent 배열)

#### BFS 계열
1) BFS 기본 (레벨 탐색)
2) **격자 BFS** (4방/8방/벽)
3) **최단거리 BFS** (단일 출발)
4) **멀티 소스 BFS** (여러 출발점)
5) **0-1 BFS / 상태 그래프** (가중치 0/1)

### 연결 원칙
- Graph 챕터는 **구조 이해** 중심
- Algorithms 챕터는 **문제 유형/패턴** 중심
- 같은 예제 재사용을 피하고 **문제 유형별 그래프**로 분리

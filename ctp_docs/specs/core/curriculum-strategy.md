# Curriculum Strategy (Refactor 이후)

## 목적
- 무지성 나열을 피하고 **선수지식/난이도/문제 대비** 기준으로 Chapter Topics을 재배치한다.
- 리팩토링 완료 이후의 시각화 품질을 전제로 커리큘럼을 구성한다.

---

## 구성 원칙
1) **선수지식 흐름**
   - 자료구조 → 기본 탐색 → 응용(패턴/문제 유형) 순서
2) **난이도 계단화**
   - 쉬운 개념 → 중간 응용 → 고난도 패턴
3) **코딩테스트 대비**
   - Baekjoon 대표 문제 유형과 매칭
4) **교재형 구조**
   - 개념 → 시각화 → 코드 패턴 → 대표 문제
5) **코드-시각화 학습 루프**
   - 사용자가 코드를 바꾸면 **시각화/출력이 즉시 반영**되는 구조를 유지
   - 코드 템플릿은 **짧고 명확**하며, 출력/규약 주석이 포함되어야 함

---

## 세션별 재구성 방향

### Algorithms
- Sorting → Binary Search → DFS → BFS(+Grid) → Shortest Path → Advanced Graph → DP 순서 유지
- Grid Traversal은 BFS 기본 이후 응용 챕터로 배치
- Advanced Graph는 Topo → MST 순서로 유지 (난이도 상승)

### Non-Linear
- Tree → Heap → Trie → Union-Find → Graph 순서로 재배치
- Graph는 가장 난이도가 높으므로 마지막에 배치
